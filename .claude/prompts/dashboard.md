# Новая функциональность

Создать главный экран

## Контекст (что уже есть)

- NestJS + Next.js + PostgreSQL + Prisma
- Авторизация (JWT), модуль категорий, модуль транзакций

## Задача

Создать главный экран приложения, в котором слева должен быть скрывающийся сайдбар (профиль) с настройками, выходом, поддержкой, навигацией по разделам

в- главное окне должно быть дропдауны в которых должны быть периоды(это календарь в котром можно выбрать дату от и до) и дропдаун Категории(можно выбрать множественные категории), помимо выбора в дашборде должен быть функционал редактирования транзакций и категорий, для
под дропдаунами добавь 2 карточки расходы и доходы в которых видна статистика за указанный период и общая сумма. Пока не кликнули на карточку, под ней должна отображаться гистограмма по месяцам где красная палочка это расход, синяя доход. При переходе в категорию, должен быть список транзакций по категории с указанием даты, суммы и назначения платежа

- категорий должна быть возможность выбора emoji иконок и текста.
- во время загрузки данных должны использоваться скелетоны
- не забудь сделать адаптивы для использования на мобильных, на адаптиве не должно быть сайд бара, профиль прячется в дропдауне в правом верхнем углу, навигация по роутам с помощью карточек на главном экране
- должны быть добавлены плавные анимации
- если возникают сетевые ошибки или какие либо другие, отображаем их во всплывающем алерте
- у нас в приложении должно быть 2 темы светлая и темная. причем они должны запоминаться при выборе и сохраняться в local-storage, до момента разлогина пользователя. дефолтная тема выбирается на основе операционной системы.

## Модель данных

Сущность продумай на основе бекенда транзакций

## Паттерн

- для бекенда следуй структуре модуля из @apps/backend/src/categories/ как образец структуры
- Взаимодействие через CQRS

## Ограничения

- При составлении плана сделать чек-лист для выполнения задачи по шагам и сохраняй план в папку @.claude/plans
- Установка в зависимости для UI компонентов разрешена.
- class-validator для DTO
- После реализации запустить сборку и провести end to end тесты
