## Аргументы

- `$0` — URL страницы (необязателен). Может быть полным (`http://localhost:3000/dashboard`) или путём (`/dashboard` — тогда базовый URL берётся из дефолта `http://localhost:3000`). **Если не передан — проверить все страницы приложения.**
- `$1` — размер вьюпорта (необязателен): `mobile` (390×844), `tablet` (768×1024) или `desktop` (1280×800). **Если не передан — проверить все три вьюпорта.**

## Режимы запуска

| `$0`       | `$1`       | Поведение                                     |
| ---------- | ---------- | --------------------------------------------- |
| передан    | передан    | одна страница, один вьюпорт                   |
| передан    | не передан | одна страница, все три вьюпорта               |
| не передан | любой      | **все страницы** приложения, все три вьюпорта |

## Шаг 0 — если `$0` не передан: обнаружить все страницы

Используй `find` для поиска страниц в директории Next.js App Router:

```bash
find apps/frontend/src/app -name "page.tsx" -o -name "page.ts"
```

Преобразуй найденные пути в URL-маршруты: убери префикс `apps/frontend/src/app` и суффикс `/page.tsx`, замени `(группы)` (в скобках) на пустую строку, `[param]` — на пример значения `1`. Корневой `page.tsx` → `/`.

Также добавь маршруты, известные из навигации: `/login`, `/register`, `/settings`, `/support`.

Дедуплицируй итоговый список.

## Шаг 1 — определить URL и список вьюпортов

Если `$0` начинается с `/` — добавь префикс `http://localhost:3000`.

Вьюпорты:
| Значение | Ширина × Высота |
|----------|-----------------|
| `mobile` | 390 × 844 |
| `tablet` | 768 × 1024 |
| `desktop` | 1280 × 800 |

Если `$1` передан — проверять только его. Если не передан — все три: **mobile → tablet → desktop**.

## Шаг 2 — открыть страницу

1. Выставь размер окна через `browser_resize`.
2. Перейди по URL через `browser_navigate`.
3. Подожди загрузки через `browser_wait_for` (до 5 с).

## Шаг 2а — авто-логин при редиректе на /login

**Если после навигации текущий URL содержит `/login` и целевая страница не была `/login` или `/register`:**

1. Заполни форму через `browser_fill_form`:
    - поле Email → `oleg@test.com`
    - поле Пароль → `password123`
2. Нажми кнопку «Войти» через `browser_click`.
3. Подожди редиректа обратно (до 5 с).
4. Перейди снова на исходный целевой URL через `browser_navigate`.
5. Подожди загрузки (до 5 с).

Авто-логин выполняется **один раз за сессию** — после успешного входа сессия сохраняется в куках браузера для всех последующих страниц.

## Шаг 3 — собрать артефакты

Параллельно (в одном сообщении) вызови:

- `browser_take_screenshot` — визуальный снимок страницы.
- `browser_snapshot` — DOM-снапшот (доступность, структура).
- `browser_console_messages` — сообщения консоли браузера.

## Шаг 4 — проанализировать

**Критические ошибки (🔴)**

- JS-ошибки или необработанные исключения в консоли.
- HTTP 4xx/5xx ответы (видны в консоли как network errors).
- Пустая страница или белый экран (нет контента в DOM).
- Сломанная вёрстка: элементы выходят за пределы вьюпорта по горизонтали.

**Предупреждения (🟡)**

- `console.warn` сообщения.
- Битые изображения (src есть, но изображение не загрузилось).
- Элементы без текстового содержимого там, где оно ожидается.
- Нечитаемый контраст или слишком мелкий шрифт.

**Успех (🟢)**

- Страница загрузилась, контент отображается, ошибок нет.

## Шаг 5 — вывести отчёт

### Формат одного вьюпорта:

```
## Визуальная проверка: <url> (<viewport>)

### Статус: 🟢 OK / 🟡 Предупреждения / 🔴 Ошибки

### Скриншот
<встроен выше>

### Находки
- 🔴 ...
- 🟡 ...
- 🟢 Страница загружена, контент отображается корректно

### Консоль
<ошибки и предупреждения или "чисто">

### DOM
<краткое описание: заголовок, секции, интерактивные элементы>
```

### Итоговая сводка (при проверке нескольких вьюпортов или страниц):

```
## Итог: <url или "все страницы">

| Страница | mobile | tablet | desktop |
|----------|--------|--------|---------|
| /        | 🟢     | 🟢     | 🟡      |
| /login   | 🟢     | 🟢     | 🟢      |
| ...      | ...    | ...    | ...     |

<общий вывод: критичные проблемы, паттерны по вьюпортам>
```

После всех отчётов закрой браузер через `browser_close`.
