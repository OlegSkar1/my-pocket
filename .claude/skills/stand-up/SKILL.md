## Задача

Собери активность в репозитории за последние 24 часа, сформируй краткий стенд-ап для руководства и сохрани его в отдельный файл.

## Шаг 1 — Сбор данных

Выполни параллельно:

```bash
git log --since="24 hours ago" --all --oneline --no-merges
git log --since="24 hours ago" --all --no-merges --stat --format="%H %s"
git branch --show-current
```

Если коммитов нет — сообщи: «За последние 24 часа коммитов не было» и завершай.

## Шаг 2 — Анализ изменений

По коммитам и статистике файлов ответь на два вопроса:

1. **Что было сделано в целом?** — сформулируй общую суть работы за день одним-двумя предложениями, без перечисления каждого коммита.
2. **Какую ценность это даёт?** — что стало возможным, что улучшилось или что было устранено.

Не перечисляй коммиты и файлы — думай о результате, а не о процессе.

## Шаг 3 — Сохранение и вывод

Определи сегодняшнюю дату в формате `YYYY-MM-DD` и сохрани стенд-ап в файл `standups/YYYY-MM-DD.md`.

Если файл на эту дату уже существует — перезапиши его (повторный запуск обновляет стенд-ап).

Формат файла:

```
## Стенд-ап — <дата>

<Абзац 1: общая суть того, что было сделано — без деталей реализации.>

<Абзац 2: на что это влияет или что стало возможным благодаря этой работе.>
```

После сохранения выведи содержимое файла и сообщи путь к нему.

**Правила формулировок:**

- Два коротких абзаца, без списков и заголовков.
- Пиши для нетехнического руководства: без жаргона, названий функций, файлов и модулей.
- Формулируй результат, а не действие: «Налажена документация API» вместо «Добавлены JSDoc-комментарии».
- Если изменения чисто технические — объясни их ценность простыми словами.
