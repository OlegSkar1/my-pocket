# Пример хорошего PR

## Заголовок

```
feat(transactions): добавить фильтрацию по дате и категории
```

## Тело

```markdown
## Что реализовано

### Backend
- Добавлены query-параметры `dateFrom`, `dateTo`, `categoryId` в `GET /api/transactions`
- Обновлён `TransactionsRepository.findMany` — поддержка фильтрации через Prisma `where`

### Frontend
- Добавлена панель фильтров на странице транзакций
- Фильтры сохраняются в URL-параметрах

## Новые API-эндпоинты

| Метод | Путь | Назначение |
|-------|------|-----------|
| GET | /api/transactions?dateFrom=&dateTo=&categoryId= | Список транзакций с фильтрацией |

## Тест-план

- [ ] Открыть страницу транзакций — список отображается без фильтров
- [ ] Выбрать диапазон дат — список фильтруется
- [ ] Выбрать категорию — список фильтруется
- [ ] Комбинировать фильтры — работают вместе
- [ ] Обновить страницу — фильтры восстанавливаются из URL
```
