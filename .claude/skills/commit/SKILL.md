## Подготовка

1. Проверь текущую ветку: `git branch --show-current`. Если `main` — **создай ветку автоматически** перед коммитом:
   - Изучи `git diff --stat` и `git status`, чтобы понять характер изменений.
   - Подбери префикс (`feat/`, `fix/`, `refactor/`, `docs/`, `chore/`) и краткое имя на английском в kebab-case, отражающее суть изменений.
   - Создай ветку: `git checkout -b <prefix>/<name>` (например, `feat/expense-filter`, `docs/readme-setup`).
   - Сообщи пользователю, какую ветку создал.

2. Выполни параллельно:
   - `git status` — незакоммиченные изменения
   - `git diff` — что именно изменилось (staged + unstaged)
   - `git log --oneline -5` — контекст последних коммитов

## Формат коммита

```
<type>(<scope>): <описание на русском языке>
```

Описание — **строчными буквами**, без точки в конце.

### Типы

| Тип        | Когда                               |
| ---------- | ----------------------------------- |
| `feat`     | новая функциональность              |
| `fix`      | исправление бага                    |
| `refactor` | рефакторинг без изменения поведения |
| `docs`     | только документация                 |
| `chore`    | конфиги, зависимости, инструменты   |
| `test`     | тесты                               |
| `style`    | форматирование, пробелы             |

### Скоупы

| Скоуп          | Область                 |
| -------------- | ----------------------- |
| `frontend`     | `apps/frontend`         |
| `backend`      | `apps/backend`          |
| `shared`       | `packages/shared-types` |
| `auth`         | модуль аутентификации   |
| `categories`   | модуль категорий        |
| `transactions` | модуль транзакций       |

Скоуп можно опустить, если изменения затрагивают весь репозиторий. Breaking change: `feat(auth)!:` + описание в теле.

### Примеры

```
feat(transactions): добавить фильтрацию по дате
fix(auth): исправить ошибку истечения токена при рефреше
refactor(frontend): вынести константы роутов в shared/config
docs: обновить README — инструкция по первичной настройке
chore: обновить зависимости до актуальных версий
```

## Создание коммита

Добавь нужные файлы в staging (предпочитай конкретные файлы, не `git add .`), затем создай коммит через heredoc:

```bash
git commit -m "$(cat <<'EOF'
feat(scope): описание
EOF
)"
```

После коммита выполни `git status` для подтверждения.

## Справка: именование веток

Если ветки ещё нет, подскажи правила:

| Префикс     | Когда                  |
| ----------- | ---------------------- |
| `feat/`     | новая функциональность |
| `fix/`      | исправление бага       |
| `refactor/` | рефакторинг            |
| `docs/`     | только документация    |
| `chore/`    | конфиги, зависимости   |

Примеры: `feat/dashboard`, `fix/auth-token-refresh`, `docs/readme-setup`.
