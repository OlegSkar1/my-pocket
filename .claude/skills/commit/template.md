# Формат коммита

## Типы

| Тип        | Когда                               |
| ---------- | ----------------------------------- |
| `feat`     | новая функциональность              |
| `fix`      | исправление бага                    |
| `refactor` | рефакторинг без изменения поведения |
| `docs`     | только документация                 |
| `chore`    | конфиги, зависимости, инструменты   |
| `test`     | тесты                               |
| `style`    | форматирование, пробелы             |

## Скоупы

| Скоуп          | Область                 |
| -------------- | ----------------------- |
| `frontend`     | `apps/frontend`         |
| `backend`      | `apps/backend`          |
| `shared`       | `packages/shared-types` |
| `auth`         | модуль аутентификации   |
| `categories`   | модуль категорий        |
| `transactions` | модуль транзакций       |

Скоуп можно опустить, если изменения затрагивают весь репозиторий. Breaking change: `feat(auth)!:` + описание в теле.

## Примеры

```
feat(transactions): добавить фильтрацию по дате
fix(auth): исправить ошибку истечения токена при рефреше
refactor(frontend): вынести константы роутов в shared/config
docs: обновить README — инструкция по первичной настройке
chore: обновить зависимости до актуальных версий
```

## Именование веток

| Префикс     | Когда                  |
| ----------- | ---------------------- |
| `feat/`     | новая функциональность |
| `fix/`      | исправление бага       |
| `refactor/` | рефакторинг            |
| `docs/`     | только документация    |
| `chore/`    | конфиги, зависимости   |

Примеры: `feat/dashboard`, `fix/auth-token-refresh`, `docs/readme-setup`.
